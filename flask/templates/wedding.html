爬取大众点评广州所有婚纱摄影店铺信息&数据分析


数据分析 数据内容


爬取大众点评的过程，主要遇到以下几个问题：

1.大众点评对评论内容进行了加密，且有两种不同的加密方法，每过一段时间就更换

2.当浏览页面的过程中，有一定几率跳转到验证中心，需要拖动验证条进行验证

3.大众点评有很强的反爬机制，当访问频率超过某个值时，会使cookie无效导致403Error，或直接禁IP

下面主要对上述遇到的问题进行分析



首先是加密，用以下评论为例


html源码长酱紫


可以看到，很多文字都被带有不同的class值的svgmtsi标签进行了替代


在顶部找到了这个css

打开就是所有这些class，且每个background带有两个px参数


在此css文件中找到该处


打开这个svg，里面是多行文字


查看这个svg的html源码，主要由path和textPath两种结构组成



下面试图用最开始的评论例子来解析整个过程


第一个“理”字对应<svgmtsi class="ldutj"></svgmtsi>

class的值为ldutj

在css文件里面找到.ldutj{background:-28.0px -1647.0px;}

用1647 + 23 = 1670

在svg文件里面找到<path id="41" d="M0 1670 H600"/>，1670所对应的id为41

在svg文件里面找到<textPath xlink:href="#41" textLength="504">赏塔理袭响脂植欧争私冤竖指锹疼翠彻应惭卸缘回属康肚臂输烂逆最扬休倡漫示冲</textPath>

再回到css文件的.ldutj{background:-28.0px -1647.0px;}，用28 / 14 = 2

在赏塔理袭响脂植欧争私冤竖指锹疼翠彻应惭卸缘回属康肚臂输烂逆最扬休倡漫示冲中，从左到右索引为2（从0开始）的字，即为“理”字


上述解析过程关键代码实现：

path = {}
for item in svg_path:
    path[int(item[1])] = int(item[0])

lines = {}
for item in svg_textPath:
    lines[int(item[0])] = item[1]

css_class = [(key_letter + cls_name, int(x), int(y)) for cls_name,x,y in css_class]
resDic = {}
for item in css_class:
    tmp = item[2] + 23
    index = item[1] / 14
    word = lines[path[tmp]][int(index): int(index)+1]
    resDic[item[0]] = word

这样就将svg的每一行的每一个字和评论的svgmtsi标签形成了映射，在读取评论页面内容时直接转换即可
